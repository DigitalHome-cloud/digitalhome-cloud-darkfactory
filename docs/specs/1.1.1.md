# 1. Overview

This specification defines the Electrical domain architecture using an
ontology‑first and modular approach:

-   The **Global Core T-BOX** contains all canonical electrical classes
    and properties.
-   **Norm modules** extend the Core using subclasses and constraints
    only.
-   Each module covers exactly one conceptual category (Electrical).
-   Modules do not introduce new base concepts.

Norm modules covered: - dhc-nfc14100-electrical -
dhc-nfc15100-electrical

------------------------------------------------------------------------

# 2. Global Core T-BOX (Electrical)

## 2.1 Core Classes

### dhc:ElectricalTechnicalSpace

Subclass of dhc:Space\
Generic technical space hosting the core electrical installation.

### dhc:EnergyDelivery

Root entry point of electrical topology. Represents contractual +
physical energy supply.

### dhc:EnergyMeter

Metering device (compteur).

### dhc:EmergencyDisconnect

Emergency/general disconnect device.

### dhc:DistributionBoard

Electrical board/cabinet hosting protection and distribution.

### dhc:Distribution

Intermediate distribution node (junction box / branching point).

------------------------------------------------------------------------

## 2.2 Core Properties

### Energy supply properties (on dhc:EnergyDelivery)

-   dhc:energyProviderName (string)
-   dhc:currentType (SinglePhase \| ThreePhase)
-   dhc:neutralSystem (Star)
-   dhc:contractedPowerKVA (decimal)

### Topology

-   dhc:feeds (upstream → downstream relationship)

### Circuit

-   dhc:phase (integer: 1 \| 2 \| 3)

### DistributionBoard

-   dhc:distributionBoardType (CommunicationBoard \| ACBoard \| DCBoard)

### Wiring

-   dhc:wiring (string, e.g. "3G2.5", "5G6")

### Installation location

-   dhc:locatedIn (component → dhc:Space)

# 3. Module: dhc-nfc14100-electrical

Scope: Delivery boundary chain (EnergyDelivery → Meter → Emergency
Disconnect).

## 3.1 Subclasses

-   dhc-nfc14100:NF14EnergyMeter ⊑ dhc:EnergyMeter
-   dhc-nfc14100:NF14EmergencyDisconnect ⊑ dhc:EmergencyDisconnect

## 3.2 Constraints

If module applied:

1.  Must include EnergyDelivery, NF14EnergyMeter,
    NF14EmergencyDisconnect.
2.  Must support topology chain via dhc:feeds: EnergyDelivery →
    NF14EnergyMeter → NF14EmergencyDisconnect → Main DistributionBoard.

------------------------------------------------------------------------

# 5. SmartHome Shell Electrical Skeleton

When a SmartHome is created:

Always generated: - ElectricalTechnicalSpace - EnergyDelivery - Main
DistributionBoard (ACBoard)

If NF modules applied: - NF14EnergyMeter - NF14EmergencyDisconnect - GTL
(if NFC 15-100 applied) - Canonical boundary chain via dhc:feeds

# 6. Acceptance Criteria

1.  Core contains canonical electrical concepts.
2.  Norm modules define only subclasses and constraints.
3.  SmartHome Shell generates valid electrical topology.
4.  Circuit subclasses align with NF C 15-100 Table n°3 values.
5.  Modules remain isolated per conceptual category.
