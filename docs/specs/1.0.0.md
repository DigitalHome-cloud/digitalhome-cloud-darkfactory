# Release v1.0.0 --- Semantic Foundation

**Status:** Implementation Complete\
**Previous release:** none

## Goals

-   Establish the DigitalHome semantic core (T-BOX v1.0).
-   Deliver the Modeler application with ontology visualization.
-   Deliver a minimal global Library management module.
-   Publish ontology artifacts to S3 for runtime consumption.

## Big Picture

Release 1.0 creates the semantic and component foundation of the
DigitalHome platform.

The Modeler defines ontology modules that describe the DigitalHome
domain. These ontology modules are compiled and published so that future
Designer releases can generate SmartHome instances from them.

The Library module provides a minimal global catalog of components
compatible with ontology classes.

No tenant SmartHome design functionality exists in this release.

------------------------------------------------------------------------

## Features

### Ontology Visualization (App: modeler)

Load and visualize the deployed ontology graph.

Acceptance criteria:

-   ontology loads without error
-   class graph renders
-   ontology version displayed

------------------------------------------------------------------------

### Ontology Publishing Pipeline (App: modeler)

TTL files stored in modeler repo semantic-core. Build pipeline parses
TTL and uploads compiled artifact to S3.

Acceptance criteria:

-   ontology build produces graph artifact
-   artifact uploaded to S3
-   runtime consumers can fetch artifact

------------------------------------------------------------------------

### Global Library (Minimal) (App: modeler)

Each library item must include:

-   id
-   title
-   compatible ontology classes
-   optional region
-   optional standards
-   version

Library is global only. Tenant library is out of scope. Library editing
requires Cognito group `dhc-admins`.

Acceptance criteria:

-   items can be created
-   items reload correctly
-   compatibility classes stored

## What's Included

### Ontology (T-Box v1.0.0)

- **48 classes** and **64 properties** across 8 design views: Spatial, Building, Electrical, Plumbing, Heating/HVAC, Network, Governance, Automation
- Ontology version bumped from 0.1.0 to **1.0.0** (`dhc-core.schema.ttl`)
- `owl:versionInfo` and `rdfs:label` extracted into `meta` field in `ontology-graph.json`
- E-Mobility refactor: `dhc:ElectricCar`, `dhc:ElectricBicycle`, `dhc:ElectricScooter` removed as T-Box classes; replaced by `dhc:vehicleType` datatype property on `dhc:ElectricVehicle` (vehicle types are now A-Box instances)

### Ontology Visualization (modeler)

- 3D force-directed graph with design-view color coding
- Three-panel layout: sidebar (class browser) + 3D canvas + inspector
- **Version badge** ("Ontology v1.0.0") displayed in the sidebar header
- View filtering, property toggle, node selection with camera focus

### Ontology Publishing Pipeline (modeler)

- `yarn parse-ontology` — parses TTL into `src/data/ontology-graph.json`
- `yarn publish-ontology` — uploads `ontology-graph.json` and `context.jsonld` to S3
- S3 paths: `public/ontology/v1.0.0/` (versioned) and `public/ontology/latest/` (alias)
- Amplify build (`amplify.yml`) runs `parse-ontology` before `build`
- S3 bucket structure documented in ADR 0010

### Authentication (modeler)

- Cognito integration using the shared portal backend (same User Pool, same AppSync API)
- `AuthContext` with SSR-safe session loading (`getCurrentUser` + `fetchAuthSession`)
- Sign-in page (`/signin/`) with `@aws-amplify/ui-react` Authenticator component
- SSO: signing in on the portal carries over to the modeler (shared Cognito Hosted UI)
- Header shows DEMO pill when unauthenticated, user name when signed in

### Global Library (modeler + portal backend)

- `LibraryItem` GraphQL type added to portal's AppSync schema with group-based auth:
  - `dhc-admins` can create, update, delete
  - Any authenticated user can read
- Library page (`/library/`) with table listing all items
- Create/edit form with ontology class multi-select (grouped by design view)
- Fields: title, version, compatible classes, region, standards, description
- DynamoDB table auto-created by Amplify

### UX & Internationalization

- Header: Modeler, Library, Portal navigation links
- User pill with DEMO/authenticated state and Sign In/Sign Out
- Language switcher: EN / DE / FR (matching portal pattern)
- Translation files for all three languages (`src/locales/{en,de,fr}/common.json`)

### Infrastructure & Security

- Modeler version bumped from 0.2.0 to **1.0.0** (`package.json`)
- `.gitignore` hardened: `amplify/` directory excluded (contains `team-provider-info.json` with AWS account IDs/ARNs), `.graphqlconfig.yml`, `.vscode/`
- `aws-exports.deployment.js` is env-var-driven (safe to commit); `aws-exports.js` and `.env.development` are gitignored
- ADR 0010: S3 bucket structure for global (`public/ontology/`, `public/library/`) and tenant (`public/smarthomes/{id}/`) content
